# All-in-One Lead Bot

Мультиканальный бот для приёма заявок из мессенджеров.

Проект помогает малому бизнесу не терять обращения клиентов: бот собирает заявку через Telegram или VK, сохраняет её в Google Sheets или SQLite и отправляет уведомление менеджеру в выбранный канал — Telegram, MAX или VK.

## Что умеет проект

- Принимает заявки от клиентов через Telegram.
- Принимает заявки от клиентов через VK.
- Ведёт клиента по короткому сценарию:
    - имя;
    - телефон;
    - тип обращения;
    - данные автомобиля;
    - описание запроса;
    - подтверждение отправки.
- Поддерживает типы обращений:
    - запчасти;
    - ремонт;
    - другой вопрос.
- Нормализует российские номера телефонов к единому формату.
- Распознаёт VIN или свободное описание автомобиля.
- Сохраняет заявки в Google Sheets.
- Может работать с SQLite вместо Google Sheets.
- Отправляет уведомления менеджеру в Telegram, MAX или VK.
- Защищает заявку от потери: сначала сохраняет данные, потом отправляет уведомление.
- Разделяет бизнес-логику, мессенджеры, хранилище и уведомления по слоям.

## Для чего это нужно

Проект решает типичную проблему малого бизнеса: заявки приходят из разных каналов, теряются в переписках, менеджер забывает ответить, а владелец не видит общей картины.

Бот собирает обращения в единую структуру и складывает их в таблицу или базу данных. Менеджер сразу получает уведомление о новой заявке.

Пример сценария:

1. Клиент пишет боту.
2. Бот задаёт несколько вопросов.
3. Клиент подтверждает заявку.
4. Заявка сохраняется в Google Sheets или SQLite.
5. Менеджер получает уведомление.
6. Заявка получает номер и статус.

## Основные слои

### core/

Ядро проекта.

Здесь находится бизнес-логика, которая не зависит от конкретного мессенджера, базы данных или канала уведомлений.

В ядре описаны:

- модель заявки `Lead`;
- сервис обработки заявки `LeadService`;
- нормализация телефона;
- разбор VIN и данных автомобиля.

### adapters/

Транспортный слой.

Адаптеры отвечают за общение с пользователем в конкретном мессенджере.

Сейчас реализованы:

- `TelegramAdapter` — работа через Telegram и aiogram 3.x;
- `VkAdapter` — работа через VK Bots Long Poll API.

Адаптер собирает данные от клиента и передаёт готовую заявку в `LeadService`.

### storage/

Слой хранения данных.

Сейчас поддерживаются:

- Google Sheets;
- SQLite.

Хранилище можно заменить без изменения бизнес-логики и адаптеров.

### notify/

Слой уведомлений менеджера.

Сейчас поддерживаются уведомления в:

- Telegram;
- MAX;
- VK.

Если уведомление не отправилось, заявка всё равно остаётся сохранённой.

## Модель заявки

Заявка содержит:

- дату и время создания;
- источник заявки;
- имя клиента;
- телефон;
- тип обращения;
- марку автомобиля;
- модель автомобиля;
- год выпуска;
- VIN;
- текст запроса;
- статус;
- номер заявки;
- технический ID пользователя в мессенджере.

Для Google Sheets предусмотрены отдельные колонки под заявки и клиентскую базу.

## Быстрый старт

```bash
python3.12 -m venv .venv
source .venv/bin/activate

# Windows:
# .venv\Scripts\activate

pip install -r requirements.txt
cp .env.example .env
python run.py
```

Перед запуском нужно заполнить `.env`.

## Как работает сценарий заявки

### Telegram

Пользователь запускает бота командой `/start`.

Бот спрашивает:

1. Имя.
2. Телефон.
3. Тип обращения.
4. Данные автомобиля.
5. Описание запроса.
6. Подтверждение заявки.

После подтверждения заявка сохраняется и менеджеру отправляется уведомление.

### VK

Пользователь пишет в сообщения сообщества.

Бот ведёт похожий сценарий:

1. Имя.
2. Телефон.
3. Тип обращения.
4. Данные автомобиля.
5. Описание запроса.
6. Подтверждение отправки.

VK-адаптер использует Long Poll и хранит состояние диалога в памяти процесса.

## Особенности реализации

- Бизнес-логика не зависит от Telegram, VK, Google Sheets или SQLite.
- Все секреты хранятся в `.env`.
- Пользовательский ввод экранируется перед отправкой сообщений.
- Телефоны приводятся к единому формату.
- VIN распознаётся отдельно от свободного описания автомобиля.
- Уведомление менеджера не является критической операцией: если оно не отправилось, заявка не теряется.
- Новые мессенджеры, хранилища и каналы уведомлений можно добавлять через отдельные адаптеры.
